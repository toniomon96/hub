# obsidian-writer

## Path conventions

| Content | Path | Notes |
|---|---|---|
| Daily briefing | `/briefings/YYYY-MM-DD.md` | One per day; overwrite on regenerate |
| Meeting note | `/meetings/{domain}/YYYY-MM-DD-{slug}.md` | domain ∈ family/personal/hobby/client/omnexus/dse/misc |
| Decision log | `/decisions/{project-slug}.md` | APPEND-only — never rewrite history |
| Inbox item | `/inbox/YYYY-MM-DD-{hash}.md` | hash = first 8 of contentHash |
| Project doc | `/projects/{project-slug}/{name}.md` | |
| Journal | `/journal/YYYY/MM/YYYY-MM-DD.md` | |

## Frontmatter (mandatory on every write)

```yaml
---
created: ISO-8601
source: cli|granola|plaud|superwhisper|martin|manual|nightly-brief
domain: family|personal|hobby|client|omnexus|dse|misc
tags: [tag1, tag2]
runId: <ulid>      # if generated by an agent run
captureId: <ulid>  # if from the capture pipeline
---
```

## Decision log rules (LOAD-BEARING)

- **Obsidian-first, then reconcile to Notion.** Write the decision to the
  Obsidian decision log immediately. Then attempt the Notion Decisions DB
  write. If Notion write fails, log a `notion_reconcile_pending` event;
  a periodic job (V1) reconciles.
- Decision entries use the format:
  ```
  ## YYYY-MM-DD — <short title>
  **Decision:** ...
  **Context:** ...
  **Alternatives considered:** ...
  **Rationale:** ...
  **runId:** <ulid>
  ```
- NEVER edit a previous decision entry. Add a new dated entry referencing
  the prior one.

## Tag discipline

- One domain tag (matches frontmatter).
- Optional project tag: `#omnexus`, `#dse`, etc.
- Optional state tag: `#open`, `#blocked`, `#waiting`, `#done`.
- No more than 5 tags per note.
