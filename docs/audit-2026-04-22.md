## Four Mandatory Questions

### 1. Has anything drifted?

**a. adversarialNote: schema theater (§VIII)**
`schema.ts:68` defines `adversarialNote` with comment "Phase 10: adversarial gate." `apps/server/src/api.ts:1045` exposes it. `run.ts` never wrote it. The column was always null. The API consumer in `apps/web/src/api.ts:12` reads a value that never existed. "Phase 10" is not in the ROADMAP. The schema + API surface created the appearance of §VIII compliance; none existed. **Fixed in this audit.**

**b. Desktop Commander in default MCP scope (§X)**
The ethos demands "Scoped credentials per tool." ROADMAP v0.4 #11 acknowledged the gap. Every agent run including the `system` scope granted the model unrestricted shell access. §X is unambiguous, the fix was acknowledged, the fix was small, and it had not shipped. **Fixed in this audit.**

**c. R2/R3 runs without consent gate (§V)**
ARCHITECTURE.md defines R2 as "Confirm prompt" and R3 as "y/N + 60s timeout + audit log." `run.ts` applied none of this. permissionTier is set by the caller and logged; nothing enforces it. The mcp_consents table is unwired. Full gating deferred to v0.6. **Partially mitigated in this audit** (quiet-hours gate added as first code enforcement).

**d. Sensitivity detection open by default (§X)**
`router.ts:detectSensitivity()` returned 'low' when `HUB_SENSITIVITY_PATTERNS` was empty. On a fresh install, nothing was ever flagged as sensitive. The privacy guarantee was only as strong as the pattern list. **Fixed in this audit** (base patterns added; prod warning added).

**e. isQuietHour() missing from run.ts (§XII, §XIV)**
`ingest.ts:isQuietHour()` existed and guarded capture inbox writes. `run.ts` had no call to it. An R3 run at 2am proceeded. The code that existed for quiet hours protected the wrong surface. **Fixed in this audit.**

### 3. What's aspirational (prose with no code)?

In descending order of leverage, **as of this audit date**:

1. **§IV semantic memory** (still aspirational): "Every conversation, decision, email, commit is queryable substrate." The substrate exists (SQLite captures) but is not queryable semantically. FTS is unimplemented. sqlite-vec is not installed. Brief has no memory injection. Addressed in v0.6 — highest-leverage item still outstanding.

2. **§V full R2/R3 consent gates**: permissionTier is logged but enforced only at the quiet-hours level (added this audit). The mcp_consents consent-prompt gate does not exist until v0.6 #6.

3. **§XI plural self population**: Template sections, no behavioral updates from observations. Manual editing is the only path until theorize.md has data to work with.

4. **§III Socratic mode**: "The right response is not an answer but a question sharp enough to collapse my confusion." No `--socratic` flag or prompt variant exists.

5. **§II quantitative augmentation test**: The annual-review.md asks the question but relies on self-report, not data. No systematic signal measures whether the unaided Toni is sharpening or dulling.

---

### 4. Single weakest link threatening §II (augmentation over substitution)?

**§IV — Memory as substrate.**

The ethos says: "Memory is not a feature of the system. Memory is the system." The current system has:
- `context.md`: does not exist on disk — bootstrapped empty on first use
- `embeddings` table: zero rows, no semantic search
- Briefings: summaries with no capture body injection until v0.6 #5
- `rawContentRef`: stores a path pointer to content, not content itself

Every session is blind to every prior session. The theorize prompt forms hypotheses; the feedback-review closes a loop; the provocation prompt names patterns — but none of these can do their jobs without memory of what actually happened. provocation.md reads `## Commitments` and finds empty comments. theorize.md fetches 60 days of feedback and finds near-zero rows. The entire intelligence layer runs on an empty context.

This is the only gap that, left unfixed, guarantees substitution over augmentation: Hub responds to what Toni says *right now* rather than who Toni is *over time*. That is not augmentation — that is a very expensive one-shot assistant.

v0.6 addresses this. v0.4 and v0.5 do not. The gap is known and scheduled. It is the gap.
